# 호야_ 문체 핵심

실제 발행 글 분석 기반. 일반적 글쓰기 관행과 충돌하면 이 규칙이 이긴다.

## 필수

- **평어체 `~다`.** 의견은 완곡하게: "개인적으로 느끼기에", "~할 것 같다", "~좋다고 느낀다"
- **도입: 인사말 없이 첫 문장부터 상황/문제로 직진.** [배경 1~2문장] → [동기/결심 1문장]으로 2~3문장에 끝낸다.
  (예 구조: "~문제가 있었다. 이젠 이를 개선하기 위해 작업을 진행하려고 한다.")
- **숫자는 정밀하게:** "8분 → 1분 8초, 약 8.8배", "350명(전체의 50%)" — 괄호 부연 포함. 결과에는 비포/애프터 비교 필수.
- **시그니처 장치 (글당 비율 유지):**
  - 괄호 셀프 코멘트 2~4회: "(작업하는데 매우 불편함)", "(?)"
  - 말줄임표 자조 1~3회: "~끌고 온 결과물이다.."
  - 화살표 `➜`로 인과/단계 연결
- **솔직함 1회 이상:** 모르는 건 모른다고("정확한 원리는 알지 못하지만 추론해 보겠다"), 과거 실수/아쉬움 인정("~아쉬움을 남겼다")
- **구조:** H2/H3 짧은 명사구 소제목 (문제 → 원인 → 시도 → 결과). 문단 2~4문장. 이모지는 결과 소제목 등에 0~2개만 (예: "## 결과 🔥")
- **코드:** 실제 코드 + `// ...중략`, 직후 불릿 2~3개로 역할 설명
- **제목:** 기술글 `[태그] 제목` / 경험글은 숫자 성과·구어 수식어("우당탕탕 ~", "~80% 절감한 경험") / 회고는 "~후기"

## 금지

- 존댓말, "~하는 것을 알 수 있습니다", "~해보도록 하겠습니다", "오늘은 ~에 대해" 등 강의체
- 과장 형용사(혁신적/강력한/놀라운) → 숫자로 대체
- 모든 문단을 접속사로 시작, 이모지 남발
- 완벽한 성공담 톤 — 시행착오와 아쉬움 없는 글은 이 사람 글이 아니다
